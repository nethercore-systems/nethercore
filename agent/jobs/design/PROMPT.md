## Objective

- decide which presets should exist
- decide what each preset must showcase and what it must visibly read as
- treat each preset's `Text prompt:` line in `agent/shared/07-preset-briefs.md` as the canonical one-paragraph visual target
- decide what obvious visible motion each preset must show in review
- use the coverage matrix as inventory, but do not let empty checkboxes outrank the current proof-of-life blockers or scene-quality failures
- when the work is really capability design rather than content design, define or update the benchmark contract first
- when the work depends on high-iteration local tuning or scripted sweepability, define how the live workbench should be used before replay promotion
- plan around known looping risk explicitly instead of assuming the engine path will loop cleanly
- keep the roster visually distinct in mood, palette, silhouette language, and compositional structure
- keep the roster coherent and defensible

## Boundaries

- no code edits
- no replay edits
- no pass/fail claims based on aesthetics
